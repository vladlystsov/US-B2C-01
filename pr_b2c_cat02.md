## Summary

**US-CAT-02: Текстовый поиск товаров**

Покупатель вводит текст в строку поиска — B2C проксирует запрос в B2B, который ищет по title и description.

## Реализация

**Endpoint:** `GET /api/v1/products?search=...`

**Валидация:**
- Минимум 3 символа → 400
- Максимум 255 символов → 400
- Спецсимволы (%, _, ') пробрасываются в B2B без поломки

**Совместимость:**
- Поиск совместим с фильтрами: `?search=iPhone&category_id=cat-1`
- Поиск совместим с сортировкой: `?search=iPhone&sort=price_asc`
- Пустой результат → 200 с `items: []`

## DoD тесты

| Тест | Сценарий |
|---|---|
| `search_returns_matching_products` | Happy path: результаты по title/description |
| `short_query_returns_400` | Запрос < 3 символов → 400 |
| `special_chars_do_not_break_query` | %, _, ' не ломают запрос |
| `empty_results_returns_200` | Нет совпадений → 200 с пустым списком |

## ADR

**Подход:** Поиск проксируется в B2B через HTTP. B2C не имеет собственного поискового индекса.

**Альтернативы:**
1. SQL LIKE на стороне B2B (chosen) — простая реализация для MVP
2. pg_trgm — требует PostgreSQL, не для SQLite
3. Elasticsearch — избыточно для текущего масштаба

**Критерии:** Сложность реализации на MVP, достаточная релевантность для каталога.
